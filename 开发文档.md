# Cloud-Platform 轻量级云盘系统（简易权限版）软件开发需求文档

---

# 1. 引言
## 1.1 文档目的
本文档旨在明确 Cloud-Platform 轻量级云盘系统（简易权限版）的全部软件开发需求，包括功能需求、非功能需求、技术架构、部署需求等，为项目开发、测试、部署提供统一标准和依据，确保开发过程有序推进，最终交付符合预期的可运行系统。本文档面向项目开发人员、测试人员，同时适配毕设展示、个人项目落地场景，语言简洁、需求明确、可落地性强。

## 1.2 项目背景
随着个人数据量不断增长，用户对私人云盘的需求日益提升，需要一款轻量、高效、可迁移、支持大数据分析且具备简易权限认证的私人云盘系统，用于存储个人文件（图片、视频、文档等），支持上传、下载、管理等核心功能，并能通过大数据分析展示文件统计、用户行为等信息。本项目基于主流技术栈开发，实现简易权限认证，聚焦核心功能落地，同时具备容器化部署能力，可快速迁移服务器，适合作为软件工程专业毕业生毕设或个人项目。

## 1.3 项目目标
- 功能目标：实现个人云盘核心功能（文件上传/下载/管理、断点续传、秒传等），配套简易大数据分析及可视化功能，新增简易权限认证功能。
- 技术目标：基于指定技术栈开发，确保系统稳定、高效、可迁移，贴合企业级开发规范，简易权限认证模块易实现、低耦合，同时降低学习和开发成本。
- 落地目标：支持 Docker 一键部署，可在 Ubuntu 云服务器上稳定运行，大数据分析模块可正常展示统计结果，权限认证模块可正常生效。
- 展示目标：界面简洁易用，功能完整可演示，大数据可视化效果清晰，权限认证逻辑清晰，可用于毕设答辩或个人项目展示。

## 1.4 适用范围
本文档适用于 Cloud-Platform 轻量级云盘系统（简易权限版）的开发、测试、部署全流程，供开发人员明确开发范围和细节，测试人员制定测试用例，部署人员完成系统上线。

## 1.5 术语与定义
- FastAPI：基于 Python 的高性能异步 Web 框架，用于开发后端 API 接口。
- MinIO：轻量级分布式文件存储系统，用于存储用户上传的文件，支持 Docker 部署。
- Spark：轻量级大数据计算框架，用于处理用户行为日志、文件统计分析。
- Elasticsearch（ES）：轻量级搜索引擎，用于存储用户操作日志、支持快速检索。
- Kibana：大数据可视化工具，用于展示 Spark 分析结果、日志统计信息。
- Docker：容器化部署工具，用于打包系统环境和代码，实现跨服务器快速迁移。
- 断点续传：文件上传过程中中断后，可从断点位置继续上传，无需重新上传整个文件。
- 秒传：文件上传前校验 MD5 值，若系统中已存在相同 MD5 的文件，直接完成上传，无需重复存储。
- 简易权限认证：初期仅实现基础的用户身份校验、接口权限控制，区分普通用户与管理员（简化版），无需复杂角色管理。

# 2. 总体概述
## 2.1 系统概述
Cloud-Platform 轻量级云盘系统（简易权限版）是一款面向个人用户、具备简易权限认证的私人云盘系统，核心功能包括用户注册登录、简易权限管理、文件管理（上传/下载/删除/重命名）、文件夹管理、断点续传、秒传，配套大数据分析模块（文件统计、用户行为分析）及可视化大盘，支持 Docker 容器化部署，可快速迁移至任意 Ubuntu 云服务器。系统采用前后端分离架构，技术栈简洁、轻量，权限认证模块简易实现、低耦合，聚焦核心功能落地，适合毕业生开发和展示。

## 2.2 技术栈概述
本系统严格遵循指定技术栈，结合大数据分析需求和简易权限认证需求优化选型，确保技术栈适配性和可落地性，具体如下：
- 前端：Vue3 + Pinia + Axios + Websocket（用于实时展示文件上传状态、日志信息）
- 网关：Nginx（反向代理、静态资源托管、请求转发）
- 后端：Python + FastAPI（异步接口开发、业务逻辑处理）
- 高性能模块：C++（文件 MD5 计算、服务器资源监控、大文件分片处理）
- 数据库：MySQL（存储用户信息、文件元数据、分析结果、权限相关数据）
- 缓存/轻量队列：Redis（缓存文件 MD5、上传断点状态、用户会话、权限缓存）
- 分布式文件存储：MinIO（存储用户上传的所有文件，支持分布式扩容）
- 大数据存储：Elasticsearch（存储用户操作日志、文件访问日志）
- 大数据计算：Spark（日志分析、文件统计、用户行为分析）
- 大数据可视化：Kibana（展示分析结果、生成可视化图表）
- 任务调度：APScheduler（定时清理回收站、定时执行 Spark 分析任务）
- 文件处理：FFmpeg（视频转码）、Pillow（图片压缩）
- 部署：Docker + Docker Compose（一键部署所有服务，支持跨服务器迁移）
- 权限认证：JWT + 简易角色标识（区分普通用户、管理员）

## 2.3 系统架构概述
系统采用前后端分离架构，整体分为 6 层，各层独立运行、相互协作，新增简易权限认证逻辑嵌入后端服务层，确保系统高可用、可扩展，架构流程如下：
1. 用户层：用户通过浏览器访问 Vue3 前端页面，进行注册登录、文件管理、查看大数据统计等操作，管理员可额外访问简易管理页面。
2. 网关层：Nginx 接收用户请求，转发后端 API 请求，托管前端静态资源，解决跨域问题。
3. 后端服务层：FastAPI 处理业务逻辑，提供 API 接口，集成简易权限认证模块（JWT 校验、角色判断），调用 C++ 高性能模块、MinIO 分布式存储、Redis 缓存等。
4. 数据存储层：MySQL 存储核心业务数据、权限相关数据，MinIO 存储用户文件，Elasticsearch 存储日志数据，Redis 提供缓存支持（含权限缓存）。
5. 大数据处理层：Spark 从 Elasticsearch 读取日志数据，进行统计分析，将结果写入 MySQL；Kibana 读取 Elasticsearch 和 MySQL 数据，生成可视化大盘。
6. 部署层：Docker 容器化所有服务，通过 Docker Compose 一键启动，支持跨服务器迁移。

架构示意图（简化版）：用户 → 浏览器 → Nginx → Vue3 前端 / FastAPI 后端（含简易权限认证） → 数据存储层（MySQL/MinIO/ES/Redis）→ 大数据处理层（Spark/Kibana）

# 3. 功能需求
本系统功能需求分为 6 大模块，均为核心必做功能，权限认证模块初期简易实现、低耦合，聚焦个人云盘核心需求、简易权限控制和大数据分析需求，确保可落地、易开发。

## 3.1 用户模块
核心功能：实现个人用户的注册、登录，配合简易权限认证模块，区分普通用户与管理员，仅支持单用户私人云盘（用户仅能查看、管理自己的文件），管理员新增简易管理权限。
- 3.1.1 用户注册：输入用户名、密码，完成注册，用户名唯一，密码加密存储（采用 bcrypt 加密），注册后默认角色为普通用户，注册成功后自动跳转至登录页面。
- 3.1.2 用户登录：输入用户名、密码，验证通过后生成携带角色标识的 JWT 令牌，用于后续接口请求验证，登录成功后根据角色跳转至对应页面（普通用户跳转云盘首页，管理员跳转简易管理页面）。
- 3.1.3 退出登录：清除本地 JWT 令牌，跳转至登录页面。
- 3.1.4 个人信息查看：登录后可查看自己的用户名、角色标识，无需修改个人信息（简化开发）。

## 3.2 简易权限认证模块（初期简单实现）
核心功能：简易实现用户角色区分、接口权限控制，仅划分普通用户和管理员两个角色，无需复杂角色管理、权限分配，降低开发难度，适配初期需求。
- 3.2.1 角色划分：仅包含两个角色，普通用户、管理员，角色标识存储在用户表，注册默认普通用户，管理员需通过手动修改数据库角色标识配置（初期简化，无需开发后台配置页面）。
- 3.2.2 接口权限控制：基于 JWT 令牌中的角色标识，控制接口访问权限，核心规则如下：
  - 普通用户：仅能访问自身文件管理、回收站、个人信息、大数据可视化查看等接口，无法访问管理相关接口。
  - 管理员：除普通用户所有权限外，可访问简易管理接口（查看所有用户列表、查看系统整体运行统计）。
- 3.2.3 页面权限控制：前端根据 JWT 令牌中的角色标识，展示/隐藏对应页面和操作按钮（普通用户隐藏管理页面入口，管理员显示管理页面入口）。
- 3.2.4 权限校验：后端接口统一拦截，校验 JWT 令牌有效性及角色权限，无权限或令牌无效时，返回明确错误提示，禁止访问接口。

## 3.3 文件管理模块
核心功能：实现个人文件的上传、下载、删除、重命名、预览，支持文件夹管理、断点续传、秒传，文件存储至 MinIO 分布式存储，结合权限认证控制文件访问权限。
- 3.3.1 文件夹管理：支持创建文件夹、删除文件夹、重命名文件夹，文件夹可嵌套（最多支持 5 级嵌套），普通用户仅显示当前用户创建的文件夹，管理员可查看所有用户的文件夹（简易管理权限）。
- 3.3.2 单文件上传：支持上传任意类型文件（图片、视频、文档、压缩包等），最大支持 10GB 单个文件，上传前校验文件 MD5，实现秒传功能；上传过程中显示上传进度，支持取消上传。
- 3.3.3 断点续传：大文件上传时，若中途中断（浏览器关闭、网络异常），再次上传同一文件时，可从断点位置继续上传，无需重新上传整个文件（基于 Redis 记录断点位置）。
- 3.3.4 文件下载：支持单个文件下载、多个文件打包下载，下载时显示下载进度，支持取消下载；普通用户仅能下载自身文件，管理员可下载所有用户文件（简易管理权限）。
- 3.3.5 文件删除：支持单个/多个文件删除，删除后文件进入回收站，未彻底删除；普通用户仅能删除自身文件，管理员可删除所有用户文件（简易管理权限）。
- 3.3.6 文件重命名：支持修改文件名称，修改后文件元数据同步更新至 MySQL 和 MinIO；普通用户仅能重命名自身文件，管理员可重命名所有用户文件（简易管理权限）。
- 3.3.7 文件预览：支持图片（JPG、PNG、GIF 等）、文本文件（TXT、MD 等）、视频（MP4、AVI 等）在线预览，其他类型文件（如压缩包、exe）不支持预览，仅支持下载；普通用户仅能预览自身文件，管理员可预览所有用户文件（简易管理权限）。
- 3.3.8 秒传功能：文件上传前，前端计算文件 MD5，后端查询 MySQL 和 MinIO，若已存在相同 MD5 的文件，直接返回上传成功，无需重复存储文件，仅更新文件元数据（关联当前用户）。

## 3.4 回收站模块
核心功能：存储删除的文件/文件夹，支持恢复、彻底删除，定时清理过期文件，结合权限认证控制回收站操作权限。
- 3.4.1 回收站查看：普通用户仅显示当前用户删除的所有文件/文件夹，显示删除时间、文件大小、文件类型；管理员可查看所有用户删除的文件/文件夹（简易管理权限）。
- 3.4.2 文件恢复：普通用户仅能恢复自身删除的文件/文件夹，恢复后文件/文件夹回到原存储路径；管理员可恢复所有用户删除的文件/文件夹（简易管理权限）。
- 3.4.3 彻底删除：普通用户仅能彻底删除自身删除的文件/文件夹，删除后无法恢复，同时删除 MinIO 中的对应文件；管理员可彻底删除所有用户删除的文件/文件夹（简易管理权限）。
- 3.4.4 定时清理：通过 APScheduler 定时清理回收站中超过 30 天的文件/文件夹（可配置清理周期），管理员可手动触发清理操作（简易管理权限）。

## 3.5 大数据分析与可视化模块
核心功能：收集用户操作日志，通过 Spark 进行统计分析，通过 Kibana 展示可视化结果，无需复杂分析，聚焦基础统计，易开发、易展示，结合权限控制展示范围。
- 3.5.1 日志收集：收集用户注册、登录、文件上传/下载/删除/重命名、文件夹操作、权限校验等所有行为日志，存储至 Elasticsearch。
- 3.5.2 统计分析（Spark）：
  - 文件统计：今日/本周/本月上传文件数、文件总大小、文件类型分布（图片/视频/文档等）；普通用户仅查看自身数据，管理员查看所有用户汇总数据。
  - 用户行为统计：今日/本周/本月登录次数、上传/下载次数、删除文件次数；普通用户仅查看自身数据，管理员查看所有用户汇总数据。
  - 热门文件统计：根据文件访问次数，统计当前用户（普通用户）或所有用户（管理员）最常访问的 10 个文件。
- 3.5.3 可视化展示（Kibana）：
  - 趋势图：展示今日/本周/本月上传文件数、上传大小、登录次数的变化趋势；普通用户仅展示自身数据，管理员展示所有用户汇总数据。
  - 饼图：展示文件类型分布占比；普通用户仅展示自身数据，管理员展示所有用户汇总数据。
  - 排行榜：展示热门文件排行（前 10）；普通用户仅展示自身文件排行，管理员展示所有用户文件排行。
  - 统计卡片：展示文件总数、总大小、今日上传数、今日登录次数等核心指标；普通用户仅展示自身指标，管理员展示所有用户汇总指标。
- 3.5.4 定时分析：通过 APScheduler 定时执行 Spark 分析任务（每日凌晨 2 点执行），将分析结果写入 MySQL，供 Kibana 调用展示，区分普通用户与管理员数据展示范围。

## 3.6 系统监控模块
核心功能：监控服务器资源状态、系统运行状态，展示监控数据，支持异常提示（简化版，无需复杂告警），结合权限控制监控查看权限。
- 3.6.1 服务器资源监控：通过 C++ 模块采集 Ubuntu 服务器 CPU 使用率、内存使用率、磁盘使用率，实时上传至 Redis；普通用户仅查看基础监控数据，管理员查看完整监控数据（简易管理权限）。
- 3.6.2 系统运行监控：监控后端服务、MinIO、MySQL、Redis、Elasticsearch、Spark 等服务的运行状态（运行/停止）；普通用户仅查看核心服务状态，管理员查看所有服务状态（简易管理权限）。

# 4. 非功能需求
## 4.1 性能需求
- 响应速度：前端页面加载时间 ≤ 2s，接口响应时间 ≤ 500ms（文件上传/下载、权限校验除外）。
- 并发能力：支持同时 10 个用户操作（适配个人/毕设场景），无卡顿、无报错，权限校验不影响接口响应速度。
- 文件处理：单个文件上传速度 ≥ 1MB/s（取决于服务器带宽），断点续传恢复时间 ≤ 1s，秒传响应时间 ≤ 300ms。
- 大数据分析：Spark 分析任务（每日统计）执行时间 ≤ 5min，Kibana 图表加载时间 ≤ 1s。
- 权限校验：JWT 令牌校验时间 ≤ 50ms，角色权限判断时间 ≤ 30ms，不影响系统整体性能。

## 4.2 可靠性需求
- 系统稳定性：连续运行 72 小时无崩溃、无异常，文件上传/下载、权限校验过程中无数据丢失、无权限错乱。
- 数据可靠性：用户文件、用户信息、日志数据、分析结果、权限相关数据均支持持久化存储，服务器重启后数据不丢失（MinIO 支持文件备份，MySQL 支持数据备份）。
- 容错性：文件上传中断、网络异常、权限校验失败后，可正常恢复；接口请求失败后，支持重试，不影响系统整体运行，权限校验失败给出明确提示。

## 4.3 可扩展性需求
- 模块扩展：各功能模块独立设计，权限认证模块预留扩展接口，后续可新增复杂角色管理、权限分配功能，无需大规模修改现有代码。
- 存储扩展：MinIO 支持分布式扩容，后续可增加服务器节点，扩大文件存储容量；Elasticsearch 支持集群扩展，提升日志存储和检索能力。

## 4.4 可迁移性需求
基于 Docker 容器化部署，支持跨 Ubuntu 云服务器迁移，迁移过程简单，无需重新配置环境、重装依赖，仅需复制 Docker 配置文件和数据卷，一键启动即可正常运行，权限相关数据随数据卷迁移，无需额外配置。

## 4.5 易用性需求
- 前端界面：简洁、直观，操作流程清晰，权限相关操作（登录、角色区分）无感知，无需复杂学习，用户可快速上手（适配毕设演示场景）。
- 错误提示：操作失败（如注册失败、上传失败、权限不足）时，给出明确的错误提示，指导用户解决问题。
- 可视化大盘：Kibana 图表清晰、美观，核心统计数据一目了然，权限区分后的展示内容清晰，适合毕设展示。

## 4.6 安全性需求
基于简易权限认证模块，保留基础安全防护，确保用户数据、文件安全和权限安全：
- 密码安全：用户密码采用 bcrypt 加密存储，不存储明文密码。
- 接口安全：采用 JWT 令牌验证 + 角色权限校验，未登录用户、无权限用户无法访问对应接口。
- 文件安全：普通用户仅能访问、管理自己的文件，管理员可访问所有用户文件（简易管理权限），防止文件泄露。
- 防注入：接口参数进行校验，防止 SQL 注入、XSS 攻击（简化版，适配毕设场景）。
- 权限安全：JWT 令牌设置过期时间，防止令牌泄露导致权限滥用；角色标识存储在数据库，不可随意篡改。

# 5. 技术需求与部署需求
## 5.1 开发环境需求
- 前端开发环境：Node.js 16+、Vue3、Pinia、Axios、Element Plus（UI 组件库）。
- 后端开发环境：Python 3.9+、FastAPI、pymysql、redis、aiofiles、python-multipart、pyjwt（JWT 生成与校验）。
- C++ 开发环境：GCC 9.0+（Windows 可使用 MinGW），用于开发高性能模块。
- 数据库环境：MySQL 8.0、Redis 6.0+、Elasticsearch 7.0+。
- 大数据环境：Spark 3.0+、Kibana 7.0+（与 Elasticsearch 版本匹配）。
- 容器环境：Docker 20.0+、Docker Compose 2.0+。

## 5.2 部署环境需求
- 服务器：Ubuntu 20.04 / 22.04 云服务器（推荐 2GB 以上内存、50GB 以上磁盘，带宽 1Mbps 以上）。
- 部署方式：Docker Compose 一键部署，所有服务（前端、后端、数据库、MinIO、ES、Spark、Kibana）均容器化，权限认证模块随后端服务一同部署。
- 端口需求：开放 80（Nginx）、8000（FastAPI）、3306（MySQL）、6379（Redis）、9000（MinIO）、9200（ES）、5601（Kibana）、4040（Spark）端口。

## 5.3 技术实现要求
- 前端：采用 Vue3 组件化开发，Pinia 管理状态，Axios 发送请求，Websocket 实现实时上传进度、监控数据展示；界面采用 Element Plus 组件库，保证美观、简洁；新增角色判断逻辑，根据 JWT 令牌中的角色标识，展示/隐藏对应页面和按钮。
- 后端：FastAPI 采用异步开发，接口规范遵循 RESTful 风格，自动生成接口文档；集成 pyjwt 实现 JWT 令牌的生成与校验，新增简易权限拦截器，实现角色权限判断；C++ 模块编译为 .so 动态库，供 Python 调用；MinIO 对接实现文件存储、读取，Redis 实现缓存、断点记录、权限缓存。
- 大数据：Elasticsearch 用于日志存储（含权限相关日志），Spark 采用 Python API 开发分析任务，区分普通用户与管理员数据统计范围，Kibana 配置可视化图表，根据权限展示对应数据。
- 部署：编写 Dockerfile（前端、后端、C++ 模块）和 docker-compose.yml，实现一键启动所有服务；数据卷映射，确保数据（含权限相关数据）持久化。

# 6. 数据需求
## 6.1 数据存储需求
- MySQL：存储用户信息、文件元数据、分析结果、权限相关数据（角色标识），采用 InnoDB 引擎，确保数据可靠性。
- MinIO：存储用户上传的所有文件，支持文件分片存储、备份，确保文件不丢失。
- Redis：缓存文件 MD5、上传断点状态、用户 JWT 令牌、服务器监控数据、权限缓存，提升系统响应速度。
- Elasticsearch：存储用户操作日志、权限校验日志，支持快速检索和统计，为 Spark 分析提供数据来源。

## 6.2 数据库表结构（核心表，新增权限相关字段）
以下为核心表结构，简化设计，新增简易权限相关字段，适配简易权限版系统，可直接用于开发：

### 6.2.1 用户表（user，新增角色字段）
```sql
CREATE TABLE user (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '加密后的密码',
    role VARCHAR(20) NOT NULL DEFAULT 'user' COMMENT '角色：user（普通用户）、admin（管理员）',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
    last_login_time DATETIME DEFAULT NULL COMMENT '最后登录时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表（含简易权限）';
```

### 6.2.2 文件表（file）
```sql
CREATE TABLE file (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL COMMENT '关联用户ID',
    name VARCHAR(255) NOT NULL COMMENT '文件名称',
    path VARCHAR(500) NOT NULL COMMENT '文件在MinIO中的存储路径',
    size BIGINT NOT NULL COMMENT '文件大小（字节）',
    md5 VARCHAR(32) NOT NULL COMMENT '文件MD5值（用于秒传）',
    type VARCHAR(20) NOT NULL COMMENT '类型：file（文件）/folder（文件夹）',
    parent_id INT DEFAULT 0 COMMENT '父文件夹ID，0表示根目录',
    is_deleted TINYINT DEFAULT 0 COMMENT '是否删除：0-未删除，1-已删除（回收站）',
    delete_time DATETIME DEFAULT NULL COMMENT '删除时间',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文件表';
```

### 6.2.3 分片上传表（chunk）
```sql
CREATE TABLE chunk (
    id INT PRIMARY KEY AUTO_INCREMENT,
    md5 VARCHAR(32) NOT NULL COMMENT '文件MD5值',
    `index` INT NOT NULL COMMENT '分片索引',
    total INT NOT NULL COMMENT '总分片数',
    user_id INT NOT NULL COMMENT '关联用户ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE,
    UNIQUE KEY unique_md5_index (md5, `index`, user_id) COMMENT '防止重复上传分片'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='分片上传表';
```

### 6.2.4 分析结果表（analysis_result，新增角色关联逻辑）
```sql
CREATE TABLE analysis_result (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL COMMENT '关联用户ID（0表示所有用户汇总，供管理员查看）',
    date DATE NOT NULL COMMENT '统计日期',
    upload_count INT DEFAULT 0 COMMENT '当日上传文件数',
    upload_size BIGINT DEFAULT 0 COMMENT '当日上传文件总大小（字节）',
    download_count INT DEFAULT 0 COMMENT '当日下载次数',
    delete_count INT DEFAULT 0 COMMENT '当日删除文件次数',
    login_count INT DEFAULT 0 COMMENT '当日登录次数',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE,
    UNIQUE KEY unique_user_date (user_id, date) COMMENT '同一用户同一日期仅一条统计记录'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='Spark分析结果表（适配权限）';
```
